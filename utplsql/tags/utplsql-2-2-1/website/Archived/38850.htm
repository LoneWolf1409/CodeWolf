General Issues Steven Feuerstein sfeuerstein steven@stevenfeuerstein.com So you install utPLSQL in schema U. You grant authority to run utPLSQL to schema X. In Oracle8i, utPLSQL uses AUTHID CURRENT_USER to make sure that when X runs U.utPLSQL, it references X objects, so there should be no problem.

Prior to 8i, you would need to have synonyms set up for U (either public or private) and grants in place so that the code could be recongized from U (via dynamic PL/SQL blocks).

Or you could install utPLSQL in each schema...
