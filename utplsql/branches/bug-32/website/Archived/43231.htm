Boolean Function Versions of utAssert Procedures Chris Rimmer chrisrimmer chris@nominet.org.uk Another thing to go on my wishlist! I'd like the procedures in utAssert to be overridden with function versions which return TRUE on success and FALSE on failure. This would allow code to be structured along the lines of:

IF utAssert.eq('Very Basic Test', .....) THEN

  utAssert.eq('More Detailed Test1', ...);

  utAssert.eq('More Detailed Test2', ...);

  .......

  utAssert.eq('More Detailed Test9999', ...);

END IF;

The point being that if the basic test fails, then all of the detailed ones will fail too, giving pages and pages of superfluous error messages.

What do you think?
