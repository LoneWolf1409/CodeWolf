Test individual procedure in package Chris Rimmer chrisrimmer chris@nominet.org.uk Suppose you have a (large) package A and have written (an even bigger) test package ut_A. You make some changes to your code and find that the procedure A.a fails some of its tests. It would be nice to be able to test just this procedure while going through the debug, fix, test cycle since running all the tests on A could take some time. Obviously once you've got A.a working you would then run the whole test package.

I have hit this problem (perhaps this is an indication that my package is getting too big and complicated) but when running the tests take several minutes, this slows things up considerably.

I would see the procedure name as being an extra argument to utPLSQL.test. It would then run just ut_Setup, ut_a and ut_Teardown.

Or is there a way of achieving this already?
