Procedures considered harmful? Sten Vesterli stenv sten@vesterli.dk Inspired by the exception handling in Java, I am considering whether we would achieve better code by banning procedures from our packages.

Consider:
- A programmer doesn't have to check for exceptions when calling a subprogram
- A programmer calling a function must put the return value somewhere
- If we define as standard that every function must return a status code, the programmer calling a function MUST consider possible errors

Return values could be 0 (zero) for normal completion, 1 for warnings, 2 and higher for errors. The actual output (payload) of the function should be returned in an OUT parameter.

What do you think? Would this work? Would it lead to better code?
